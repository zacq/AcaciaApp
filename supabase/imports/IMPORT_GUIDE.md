# Animals Import Guide

## How to Import

1. Open `animals_import_template.csv` in Excel
2. Add your data below the header row (do not change column names)
3. Save as **CSV UTF-8** (File → Save As → CSV UTF-8)
4. In Supabase → Table Editor → `animals` → **Import data** → upload the CSV

---

## Field Reference

### Required Fields (cannot be blank)

| Column | Format | Allowed Values | Example |
|--------|--------|----------------|---------|
| `tag_number` | text | Unique per animal | `AV-0001` |
| `breed` | text | Any breed name | `Dorper` |
| `sex` | text | `male` / `female` | `female` |
| `birth_date` | date | YYYY-MM-DD | `2023-03-15` |

### Identity Fields

| Column | Format | Notes |
|--------|--------|-------|
| `visual_id` | text | Visual ear tag number |
| `eid` | text | RFID/Electronic ID (15-digit) |
| `name` | text | Animal name (optional) |
| `pedigree_id` | text | Stud book / registry ID |
| `registration_number` | text | Official registration |
| `scrapie_tag` | text | Scrapie/disease control tag |

### Classification Fields

| Column | Format | Allowed Values |
|--------|--------|----------------|
| `species` | text | `sheep` / `goat` / `cattle` (default: `sheep`) |
| `breeding_class` | text | `Fullblood` / `Purebred` / `Commercial` |
| `born_as` | text | `single` / `twin` / `triplet` / `quad` |
| `color_markings` | text | Free text description |
| `color_type` | text | Short label e.g. `Black Head` |
| `breed_percentage` | number | e.g. `87.5` for 87.5% |

### Date Fields

| Column | Format | Notes |
|--------|--------|-------|
| `birth_date` | YYYY-MM-DD | Required |
| `acquisition_date` | YYYY-MM-DD | Date animal entered the farm |
| `withdrawal_end_date` | YYYY-MM-DD | Only if on withdrawal |

### Parentage Fields (text fallbacks — no UUIDs needed)

| Column | Format | Notes |
|--------|--------|-------|
| `dam_sire` | text | Tag/name of dam's sire |
| `sire_sire` | text | Tag/name of sire's sire |
| `adopted_dam` | text | Tag/name of adopted dam |
| `breeder_name` | text | Name of original breeder |
| `source_country` | text | e.g. `South Africa` |
| `import_batch` | text | e.g. `BATCH-2023-A` |

> Note: `sire_id` and `dam_id` (UUID links) cannot be set via CSV import.
> Use the app to link parents after import.

### Status Fields

| Column | Allowed Values | Default |
|--------|----------------|---------|
| `status` | `active` / `sold` / `dead` / `culled` / `archived` | `active` |
| `breeding_status` | `open` / `exposed` / `pregnant` / `bred` / `retired` | *(blank)* |
| `health_status` | `healthy` / `under_treatment` / `monitor` / `critical` | `healthy` |
| `cull` | `true` / `false` | `false` |
| `withdrawal_status` | `true` / `false` | `false` |

### Physical Fields

| Column | Format | Notes |
|--------|--------|-------|
| `weight_kg` | number | e.g. `68.5` |
| `body_condition_score` | number | 1.0 – 5.0 in 0.5 steps |

### Other

| Column | Format | Notes |
|--------|--------|-------|
| `notes` | text | Free text |

---

## Do NOT include these columns
These are auto-generated by the database:

- `id` — auto UUID
- `created_at` — auto timestamp
- `updated_at` — auto timestamp
- `archived_at` — set by the app on archive
- `sire_id` — UUID FK, link via app after import
- `dam_id` — UUID FK, link via app after import
- `current_group_id` — UUID FK, assign via app after import
