# Product Requirements Document

## AcaciaVeld Ranch Operations Web App

**Version:** 1.0
**Platform:** React Web App
**Backend:** Supabase
**Brand Theme:** Green and White
**Primary Goal:** Centralize ranch activity data collection, flock management, treatment records, breeding records, reporting, and operational notes in one system.

---

## 1. Product Overview

**AcaciaVeld** is a ranch operations web app designed to help livestock breeders and ranch managers collect, organize, and act on operational data across the ranch.

The app will digitize key livestock workflows including:

* flock registration
* animal grouping
* treatment tracking
* health monitoring
* breeding records
* lambing records
* pregnancy scan tracking
* movement logging
* weight/performance tracking
* note-taking and reminders
* ranch reporting

This product should feel like a **field-ready livestock control center**, not a generic admin dashboard.

---

## 2. Product Vision

Build a web application that allows AcaciaVeld staff to:

* record livestock activities in real time
* manage animal-level and group-level data
* monitor breeding and health events
* generate useful ranch reports
* reduce paper-based errors
* improve traceability across the ranch
* support operational decision-making with accurate records

The app should be simple enough for daily use, but strong enough to become the ranch’s operational memory.

---

## 3. Core Objectives

### Primary Objectives

* Create a central ranch data system
* Make livestock records searchable and structured
* Simplify daily activity logging
* Generate operational and breeding reports
* Improve visibility into animal health, reproduction, and sales readiness

### Secondary Objectives

* Reduce data loss from manual recordkeeping
* Improve follow-up on treatments, pregnancy scans, and lambing
* Create a clean base for future automation, analytics, and mobile expansion

---

## 4. Target Users

### Ranch Owner / Admin

Needs full oversight across all operations, reports, breeding performance, medicine usage, and sales-ready stock.

### Farm Manager

Needs to supervise flock structure, movements, treatments, lambing, and team activity.

### Field Staff / Data Entry Personnel

Needs simple forms for entering daily events quickly.

### Vet / Breeding Specialist

Needs access to treatment, pregnancy scan, lambing, and breeding history.

---

## 5. Product Scope

The application will be a **React-based responsive web app** with Supabase powering:

* authentication
* PostgreSQL database
* row-level security
* storage for animal images and documents
* server functions for advanced workflows
* notifications/reminders logic where needed

## 10. Functional Flows

## 10.1 Add Animal

1. Click floating add button
2. Select “New Animal”
3. Complete form
4. Save to Supabase
5. Animal appears in flock and group

## 10.2 Log Treatment

1. Open Treatments
2. Select animal or group
3. Enter medicine + dosage + date
4. Save
5. System updates withdrawal status if needed

## 10.3 Record Breeding

1. Open Breeding
2. Select female animal
3. Assign sire
4. Enter breeding details
5. Save
6. Expected due date is auto-calculated

## 10.4 Record Lambing

1. Open Lambing
2. Select mother
3. Enter birth details
4. Save
5. Option to create offspring animal profiles immediately

## 10.5 Save Note with Reminder

1. Open Notes
2. Write note
3. Add reminder date/time
4. Save
5. Reminder stored for dashboard / notification logic

# 3. User flows

## 3.1 Authentication flow

1. User visits `/login`
2. Enters email and password
3. Supabase Auth validates credentials
4. App fetches profile and ranch role
5. Redirect to `/app/home`
6. Route guards apply permissions

---

## 3.2 Add new animal

1. User clicks floating `+`
2. Selects **New Animal**
3. Fills animal form:

   * tag number
   * breed
   * sex
   * birth date
   * source
   * group
4. Optional image upload
5. Submit
6. Save to `animals`
7. If group selected:

   * create `group_memberships` row
8. Redirect to animal profile

---

## 3.3 Record treatment

1. User clicks quick action **Treatments**
2. Select animal or group
3. Choose medicine
4. Enter dosage, route, reason
5. Enter treatment date and withdrawal days
6. Submit
7. Save to `treatments`
8. Auto-calculate withdrawal end date
9. Update animal cached withdrawal status
10. Reduce medicine quantity if linked inventory exists

---

## 3.4 Record breeding

1. User opens Breeding module
2. Select female animal
3. Select sire or enter external sire
4. Enter breeding date and mating type
5. App calculates expected due date
6. Submit
7. Save to `breeding_records`
8. Update female breeding status to `bred`

---

## 3.5 Record pregnancy scan

1. Open Pregnancy Scan
2. Select animal
3. Link breeding record if available
4. Enter scan result and date
5. Submit
6. Save to `pregnancy_scans`
7. If positive:

   * update breeding status to `pregnant`
   * store expected due date

---

## 3.6 Record lambing

1. Open Lambing
2. Select dam
3. Optionally select sire / breeding record
4. Enter birth totals and notes
5. Submit
6. Save lambing record
7. Prompt: “Create offspring records?”
8. User enters each offspring’s sex, weight, tag
9. Save offspring and optionally create animal records

---

## 3.7 Record movement

1. Open Movements
2. Select animal or group
3. Enter from-group and to-group
4. Add date and reason
5. Save
6. App updates current group in animal record
7. Closes previous `group_memberships` and creates new current membership

---

## 3.8 Save note with reminder

1. Open Notes
2. Add title and note content
3. Optional link to animal/group/module
4. Set reminder time
5. Save
6. Dashboard widget shows upcoming reminder
7. Notification system can later hook into this

---

## 3.9 Generate report

1. Open `/app/reporting`
2. Select report type
3. Apply filters:

   * date range
   * species
   * breed
   * group
4. App queries Supabase
5. Show results in table
6. User exports CSV or PDF

---

## 3.10 Archive animal

1. Open animal profile
2. Click archive
3. Select archive reason:

   * sold
   * dead
   * culled
   * transferred
4. Confirm
5. Update animal status
6. Set `archived_at`
7. Remove from active flock screens
8. Keep in archive module and reports

---

# PRD UPDATE v2.0 — PEDIGREE, REGISTRATION & CERTIFICATION ENGINE

## Overview

New Core Module: **Pedigree & Breeding Intelligence System**

This module enables:
- Full multi-generation lineage tracking (up to 5 generations)
- Breed purity validation and breed percentage calculation
- Registration number and pedigree ID management
- Certificate generation (PDF with frozen snapshot)
- Buyer-facing pedigree reports with weight trend, photos, and ancestry
- Integration with SA Stud Book, American Dorper Society, and similar registries

### pedigree_cache (NEW — performance optimization)

Avoids recomputing lineage tree on every load. Rebuild on animal create/update.

| Field        | Type               | Notes                   |
| ------------ | ------------------ | ----------------------- |
| id           | uuid PK            |                         |
| animal_id    | uuid              | Subject animal          |
| generation   | integer            | 1 = parent, 5 = 5th gen |
| ancestor_id  | uuid              | Ancestor animal ID      |
| relation_type| text               | sire / dam              |
| cached_at    | timestamptz        |                         |

Use live recursive query for accuracy. Use pedigree_cache for display speed and PDF generation.

Recursive query template (Postgres):

```sql
WITH RECURSIVE pedigree AS (
  SELECT id, sire_id, dam_id, 1 AS generation
  FROM animals
  WHERE id = :animal_id

  UNION ALL

  SELECT a.id, a.sire_id, a.dam_id, p.generation + 1
  FROM animals a
  JOIN pedigree p ON a.id = p.sire_id OR a.id = p.dam_id
  WHERE p.generation < 5
)
SELECT * FROM pedigree;
```

---

### animal_certificates (NEW)

Frozen snapshots used for PDF generation and buyer reports.

## User Flows

### FLOW 1: Register New Animal
1. User opens "Add Animal" (full-page form, not bottom sheet)
2. Fills identification fields
3. Selects Sire and Dam from searchable animal list
4. Captures up to 3 photos (front, side, rear)
5. Saves initial weight (creates animal_weights record)
6. System validates no duplicate tag number
7. Saves animal to Supabase
8. Triggers pedigree_cache generation
9. Redirects to Animal Profile

### FLOW 2: View Pedigree
1. Open Animal Profile
2. Tap "Pedigree" tab
3. System loads pedigree_cache or runs recursive query
4. Displays horizontal tree (5 generations)
5. Tapping any ancestor opens that animal's profile

### FLOW 3: Generate Certificate
1. Open Animal Profile → tap "Generate Certificate"
2. System gathers: animal details, latest weight, 5-gen pedigree, photos, breeder info
3. Freezes snapshot as JSONB
4. Generates PDF
5. Stores PDF in Supabase Storage → certificates bucket
6. Saves record in animal_certificates
7. Opens preview / share sheet

### FLOW 4: Buyer Report (Pre-Sale)
1. Select animal → "Generate Buyer Report"
2. Output includes: pedigree tree, weight trend chart, breed %, photos
3. Export as PDF

## Business Logic Notes

- Breed percentage must be derivable from ancestry chain (fullblood parents → 100%, one commercial → 50%, etc.)
- Inbreeding coefficient can be computed from pedigree_cache in future phase
- pedigree_cache should be invalidated and rebuilt whenever sire_id or dam_id on any ancestor changes
- Certificate PDFs are point-in-time snapshots — they are NEVER regenerated from live data, only new ones are issued
- All weights must be stored historically — animals.weight_kg is only a display cache
